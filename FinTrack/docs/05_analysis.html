Data analysis¶

This section describes the data analysis process for FinTrack. It covers how the cleaned bank data is analyzed to extract insights and generate reports.

Submodules¶

analysis.py module¶

05_analysis.analysis.main()¶

Main function for the financial analysis pipeline.

This function performs the following steps:

  1. Configures logging to record analysis progress and errors.

  2. Connects to a SQLite database containing bank transaction data and loads the data into a pandas DataFrame.

  3. Converts and processes date and amount columns, and computes additional columns for costs, income, and date parts (year, month, day).

  4. Categorizes transactions using a fast, vectorized string-matching approach based on predefined keyword categories.

  5. Exports uncategorized transactions (“Other” category) to a JSON file for manual review.

  6. Computes and aggregates financial summaries:

    • Yearly and monthly summaries of income, expenditure, and net balance.

    • Category-wise summaries.

    • Unique sender/receiver and purpose pairs for uncategorized transactions.

    • Yearly and monthly delta tables for expense changes.

  7. Generates and saves cumulative area plots for overall and per-category spending.

  8. Creates a comprehensive PDF report that includes:

    • Summary tables (yearly, monthly, category-wise, and details for 1999).

    • Cumulative spending plots.

    • Detailed tables for each category and keyword in 1999.

    • List of unique uncategorized transactions.

  9. Logs all major steps and outputs the location of the generated PDF report.

Note: - Requires global variables or objects such as categories, normalize_text, BUILD_DIR, ANALYSIS_DIR, and necessary imports (e.g., pandas, matplotlib, FPDF, logging, os, datetime, re, sqlite3). - Intended to be run as the main entry point for the analysis script.
