Data cleaning¶

This section describes the data cleaning process for FinTrack. It covers how raw bank data is standardized, cleaned, and imported into the database.

Modules¶

db_update.py module¶

03_data_cleaning.db_update.create_transactions_table(db_path, table_name='transactions')¶

Create the transactions table with all possible columns and a unique constraint on transaction_hash.

03_data_cleaning.db_update.detect_bank_name(filename)¶

Extract only the bank name from filename (without extension and year/suffix). E.g. ‘Bank_A -2024.csv’ or ‘Bank_A_2024.csv’ -> ‘Bank_A’

03_data_cleaning.db_update.find_header_row(file_path, sep=';', encoding='utf-8')¶

Find the header row in a CSV file by matching known column names. Returns the row index of the header.

03_data_cleaning.db_update.load_csv_with_mapping(file_path, bank_name, sep=';')¶

Load a CSV file, map its columns to unified names, clean and convert data types, and return a DataFrame ready for database insertion. Tries UTF-8 encoding first, then falls back to ISO-8859-1.

03_data_cleaning.db_update.row_hash(row)¶

Generate a hash for a row based on all available data.

03_data_cleaning.db_update.save_to_sqlite(df, db_path='C:\\public_projects\\portfolio\\FinTrack\\03_data_cleaning\\bank_statements.db', table_name='transactions')¶

Save the DataFrame into an SQLite database, ignoring duplicates. Logs the number of records saved and duplicates skipped.
